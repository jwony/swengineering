<userAccount 설명 - 김혜석 담당>

프로그램 실행시 항상! 로그인 계정을 새로 만들어야 한다.
로그인 계정은 vector형식으로 저장이 되며 프로그램 종료와 함께 데이터는 지워진다.
따라서 프로그램 실행시, NAME, ID, PASSWORD를 입력한 후 REGISTER 버튼을 누른다.
다음으로 로그인계정으로 만든 올바른 ID와 PASSWORD를 입력하면 PersonalAssistantSystem 화면으로 넘어간다.
이때 NAME은 작성안해도 된다.  
ID와 PASSWORD를 변경하고 싶을시 PersonalAssistantSystem 화면에서 ACCOUNT 버튼을 선택하면 
ChageIdAndPassword화면으로 넘어간다. 저희 시스템의 한계는 로그인 계정이 파일 입출력형식으로 저장 혹은 
불러오는 것이 안되기 때문에 처음 로그인할때 만들었던 계정은 이미 vector에서 초기화된다.
파일입출력형식으로 만들어보려고 노력했으나 입력이 제대로 되지 않아 우선 UI측면으로 프로토타입을 만들었다.
원래의 아이디어는 originalId와 originalPassword를 입력하면 이에 맞는 userAccount를 vector 콜렉션에서 찾아
newId와 newPassword로 바꾸는 것입니다. 이때 ID만 바꾸고 싶을 시 newId만 다르게 작성하고 나머지 부분은 
기존의 것과 같게 작성하시면 된다. 작성완료 후에는 다시  PersonalAssistantSystem 화면으로 넘어간다.
Logout을 원할시  PersonalAssistantSystem 화면에서 Account버튼을 누르면 UserAccount 화면이 나온다. 
logout버튼을 누르면 처음 화면인 FirstScreen화면으로 바뀐다. 

<phoneBook설명 - 서규원 담당>

mainPanel()에서 PhoneBook 버튼을 클릭하면 phoneBookFrame()가 실행되며 폰북을 관리하는 창이 뜬다.
이곳에서는 사용자가 이름, 번호, 주소를 직접 입력하여 추가할 수 있다. 
정보를 작성하고 저장 버튼을 누르면 밑에 보이는 리스트칸에 작성한 정보가 보인다. 
만약 이름, 번호, 주소 칸 중 한칸이라도 작성하지 않았다면 "이름/번호/주소가 모두 입력되지 않았습니다."라는 경고창이 뜨며 저장이 되지 않는다. 
취소를 누를 경우에는 입력했던 정보들이 사라진다. 
만약 리스트창에 있는 입력했던 정보를 삭제하고 싶다면 삭제하고 싶은 인덱스를 선택한 뒤 삭제 버튼을 누르면 삭제할 수 있다.
PhoneBookManagementTest에 대한 Unit test는 test정보를 따로 입력 한 뒤 User 벡터 안에 제대로 저장이 됐는지 확인하는 테스트이다.

<Schedule 설명 - 윤정원 담당>

schedule 버튼을 눌렀을 때 스케줄 관리 화면이 뜬다.
스케줄 관리화면의 왼쪽에 위치한 달력의 기본 값은 오늘의 날짜이고
달력 상단의 년, 월을 내가 원하는 숫자로 바꿔주면
해당 년도, 해당 달의 달력으로 이동한다.
달력의 특정 날짜를 클릭하면 오른쪽에 해당날짜의 스케쥴리스트가 보인다.
오른쪽 화면에는 오늘의 날짜와 오늘의 스케쥴 리스트가 뜬다.
이 화면에서 스케쥴 추가, 수정, 삭제가 가능하다.
오른쪽 상단에 있는 "Go Today" 버튼을 클릭하면 오늘 날짜로 돌아간다.
오른쪽 상단에 있는 "Add Schedule" 버튼을 클릭하면 스케쥴을 입력할 수 있는 팝업창이 뜬다.
팝업 창에서 시간, 제목, 내용을 입력하고 "SAVE" 버튼을 클릭하면 데이터가 저장되고 창이 자동으로 꺼진다.
"CANCEL" 버튼을 클릭하면 작업하던 내용이 중지되고 창이 꺼진다.
"Modify" 버튼을 클릭하면 스케쥴 내용을 수정할 수 있는 팝업창이 뜬다.
"Delete" 버튼을 클릭하면 해당 스케줄이 삭제된다.
